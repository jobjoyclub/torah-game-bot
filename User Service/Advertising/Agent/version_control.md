## Правила версионности

### Формат версий
- **Major.Minor.Patch** (например, 1.0.0)
- **Major**: Кардинальные изменения в функциональности
- **Minor**: Новые функции и улучшения
- **Patch**: Исправления ошибок и мелкие улучшения

### Процесс обновления
1. **Тестирование**: Новая версия тестируется на ограниченном наборе данных
2. **Валидация**: Проверка качества и эффективности
3. **Документирование**: Обновление документации
4. **Развертывание**: Постепенное внедрение в production
5. **Мониторинг**: Отслеживание результатов

### Критерии для новой версии
- **Major**: Изменение архитектуры или кардинальные улучшения
- **Minor**: Добавление новых источников или типов организаций
- **Patch**: Исправления в существующей функциональности
