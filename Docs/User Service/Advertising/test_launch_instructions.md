# Инструкции для тестового запуска агента

## Подготовка к тестированию

### 1. НАСТРОЙКА ОКРУЖЕНИЯ

#### Создать рабочую папку:
```
/Project Torah/User Service/Advertising/Test Results/
├── Pilot Test/
│   ├── results_pilot.xlsx
│   ├── progress_log.md
│   └── quality_check.md
├── Extended Test/
│   ├── results_extended.xlsx
│   ├── progress_log.md
│   └── quality_check.md
└── Full Test/
    ├── results_full.xlsx
    ├── progress_log.md
    └── quality_check.md
```

#### Подготовить инструменты:
- Excel или Google Sheets для создания таблиц
- Браузер для верификации веб-сайтов
- Доступ к социальным сетям
- Онлайн-сервисы проверки email

### 2. ПИЛОТНЫЙ ТЕСТ (1-2 дня)

#### Задача для агента:
```
Найти и верифицировать контакты еврейских организаций в следующих странах:
- United States (приоритет: New York, Los Angeles, Chicago)
- Israel (приоритет: Tel Aviv, Jerusalem, Haifa)
- Germany (приоритет: Berlin, Munich, Frankfurt)
- United Kingdom (приоритет: London, Manchester)
- Canada (приоритет: Toronto, Montreal, Vancouver)

Фокус на:
1. Религиозные общины и синагоги
2. Образовательные учреждения
3. Культурные центры

Цель: 50-100 верифицированных контактов
```

#### Инструкции для агента:
1. **Начать с официальных источников**:
   - World Jewish Congress directory
   - European Jewish Congress directory
   - Официальные сайты организаций

2. **Использовать социальные сети**:
   - Facebook: поиск по ключевым словам "Jewish community [city]"
   - LinkedIn: поиск организаций по типу "Religious Organizations"
   - Instagram: поиск хештегов #jewishcommunity, #synagogue

3. **Верифицировать каждый контакт**:
   - Проверить доступность веб-сайта
   - Проверить формат email-адреса
   - Указать источник информации
   - Зафиксировать дату проверки

4. **Структурировать результаты**:
   - Использовать шаблон из test_results_template.md
   - Заполнить все обязательные поля
   - Добавить примечания при необходимости

#### Ожидаемые результаты:
- **Количество**: 50-100 контактов
- **Стран**: 5
- **Типов организаций**: 3 (религиозные, образовательные, культурные)
- **Качество**: 90%+ актуальных email-адресов

### 3. ПРОЦЕСС ВЕРИФИКАЦИИ

#### Проверка email-адресов:
1. **Формат**: Использовать regex для проверки формата
2. **Домен**: Проверить существование домена
3. **Активность**: Проверить через онлайн-сервисы (Hunter.io, EmailHippo)
4. **Альтернативы**: Отметить, если организация предпочитает веб-формы

#### Проверка веб-сайтов:
1. **Доступность**: Проверить, что сайт загружается
2. **Актуальность**: Проверить дату последнего обновления
3. **Контент**: Убедиться, что это еврейская организация
4. **Контакты**: Проверить наличие контактной информации

#### Проверка социальных сетей:
1. **Активность**: Проверить дату последнего поста
2. **Релевантность**: Убедиться, что это еврейская организация
3. **Контакты**: Найти контактную информацию в профиле
4. **Верификация**: Проверить официальность страницы

### 4. КОНТРОЛЬ КАЧЕСТВА

#### Ежедневная проверка:
- [ ] Количество найденных контактов
- [ ] Процент актуальных email-адресов
- [ ] Разнообразие источников
- [ ] Заполненность обязательных полей

#### Еженедельная проверка:
- [ ] Географическое распределение
- [ ] Разнообразие типов организаций
- [ ] Качество примечаний
- [ ] Соответствие формату

#### Финальная проверка:
- [ ] Верификация случайной выборки (10% контактов)
- [ ] Проверка всех источников
- [ ] Анализ качества данных
- [ ] Подготовка отчета

### 7. СЛЕДУЮЩИЕ ШАГИ

#### После пилотного теста:
1. **Анализ результатов**:
   - Оценить качество данных
   - Выявить проблемы
   - Определить эффективные источники

2. **Корректировка подхода**:
   - Обновить инструкции
   - Добавить новые источники
   - Улучшить процесс верификации

3. **Планирование расширенного теста**:
   - Увеличить количество стран
   - Добавить новые типы организаций
   - Расширить временные рамки

**Готовность к запуску**: Убедитесь, что все файлы созданы, инструменты настроены, и агент понимает задачу перед началом тестирования.
